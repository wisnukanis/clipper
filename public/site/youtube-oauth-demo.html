OAuth Grant Process

User-visible flow
1 User opens app

The user signs in to Clipper Emsa Pro and chooses YouTube reconnect or publishing setup.

2 Google consent

Google shows the app name, OAuth client, and the YouTube upload permission requested by the app.

3 Callback exchange

The callback receives an authorization code and exchanges it for a refresh token.

4 Scheduled upload

GitHub Actions creates a temporary access token and uploads the final generated Short.

Sensitive And Restricted Scope Usage

YouTube only
https://www.googleapis.com/auth/youtube.upload

This scope is used only to upload the final generated short video to the authenticated user's YouTube channel.

What the app does not do

The app does not delete YouTube videos, manage comments, read private channel analytics, access Gmail, access Drive, or use unrelated Google APIs.

Refresh token handling

Refresh tokens are stored as encrypted GitHub Actions secrets or server environment variables. Access tokens are short-lived and are generated only when needed.
