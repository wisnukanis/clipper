Recording Checklist

  1. Open Google Cloud Console and show the project, app name, OAuth consent screen status, authorized domain, and OAuth client ID.
  2. Open Clipper Emsa Pro dashboard and show the YouTube platform status.
  3. Click Reconnect YouTube to start the OAuth grant flow.
  4. Show the Google consent screen and the requested YouTube upload permission.
  5. Approve access, return to the callback page, then show the dashboard preflight result with YouTube token valid.
  6. Run or show the publish workflow where the final generated clip is uploaded as a YouTube Short.

Sensitive And Restricted Scope Usage

https://www.googleapis.com/auth/youtube.upload

This scope is used only to upload the final generated short video to the authenticated user's YouTube channel. The app does not use this scope to read private channel data, delete videos, manage comments, access analytics, or perform unrelated YouTube account actions.

Offline access / refresh token

The app requests offline access so scheduled GitHub Actions workflows can create a short-lived access token when a queued video is ready to publish. Refresh tokens are stored as encrypted GitHub Actions secrets or local server environment variables. Access tokens are temporary and are not stored in the application UI.

Restricted scopes

This YouTube publishing flow does not request Gmail, Drive, Calendar, or other restricted Google API scopes. Each OAuth client in this project should be shown in the video, and each should be explained with the same upload-only purpose if it uses the YouTube upload scope.

User Flow To Narrate

  1. The user signs in to the private dashboard.
  2. The user clicks Reconnect YouTube.
  3. Google shows the app name, OAuth client, and requested YouTube upload permission.
  4. The user grants permission.
  5. The app receives an authorization code at the callback URL and exchanges it for a refresh token.
  6. The refresh token is saved securely and later used to upload generated clips selected by the user or queue.

Data Handling Statement

  • Uploaded video files are generated from source video links selected in the dashboard or queue.
  • Video title, description, tags, and captions are generated for publishing metadata.
  • OAuth tokens are used only for authentication and publishing to the user's connected YouTube channel.
  • Users can revoke app access from their Google Account permissions page at any time.
